## C. 推荐技术栈与选型理由

### 推荐最终技术栈

| 类别 | 推荐方案 | 备选 |
|---|---|---|
| **GUI 框架** | PySide6（Qt for Python） | PyQt5/6 |
| **数据可视化** | pyvista + vtk | matplotlib（简单图表） |
| **本地数据库** | SQLite | TinyDB（轻量） |
| **配置管理** | YAML + Python dataclass | JSON |
| **日志方案** | Python logging + QTextEdit 日志面板 | loguru |
| **任务执行** | QThread（Qt 原生线程） | threading（配合信号） |
| **打包发布** | PyInstaller（.spec 配置） | Nuitka |
| **测试方案** | pytest + pytest-qt | unittest |
| **代码质量** | ruff（lint）+ black（格式化）+ mypy（类型检查） | |

### 选型理由

**为什么选 PySide6：**
- Qt 生态成熟，组件丰富（树形控件、表格、日志面板等桌面组件开箱即用）
- 信号/槽机制天然适合 UI 与业务逻辑解耦
- 跨平台能力好，未来可扩展到 Windows/macOS
- 官方支持 LGPL 许可，商业友好
- 文档完善，社区活跃

**为什么选 pyvista：**
- 基于 VTK，支持 OpenFOAM 结果直接读取（OpenFOAM 输出格式兼容）
- 支持 3D 可视化（流场、压力场、温度场等）
- API 简洁，与 numpy 数组无缝衔接

**为什么选 QThread：**
- Qt 原生线程方案，与 PySide6 信号机制天然集成
- 可通过 Qt 信号实时更新 UI 进度、日志
- 避免手动管理线程同步问题

**为什么不选其他：**
- Tkinter：功能太弱，组件简陋，不适合复杂桌面应用
- wxPython：生态和文档不如 Qt，第三方组件少
- PyQt5：与 PySide6 基本相同，PySide6 是官方维护的 Python 绑定，更推荐
- multiprocessing：对桌面 GUI 场景过重，QThread 更轻量
- asyncio：对 I/O 密集任务有帮助，但 subprocess 调用已有 QThread 足够

## D. 客户端系统架构设计

```
┌─────────────────────────────────────────────────────┐
│                   UI 展示层                          │
│  （PySide6 QMainWindow / QWidget / 各类控件）         │
├─────────────────────────────────────────────────────┤
│                  应用控制层                           │
│  （QApplication / 窗口管理 / 菜单栏 / 工具栏）          │
├─────────────────────────────────────────────────────┤
│                  业务逻辑层                           │
│  ProjectManager / CaseManager / ConfigGenerator      │
│  TaskExecutor / ResultManager / TemplateManager      │
├─────────────────────────────────────────────────────┤
│                OpenFOAM 集成层                       │
│  OFCommandRunner / OFParser / OFTemplateEngine      │
├─────────────────────────────────────────────────────┤
│              Linux 本地进程调用层                     │
│  SubprocessRunner / OFEnvironment / ProcessMonitor  │
├─────────────────────────────────────────────────────┤
│                  文件管理层                           │
│  WorkspaceFS / CaseFS / ResultFS                    │
├─────────────────────────────────────────────────────┤
│                  数据存储层                           │
│  SQLite（项目元数据/任务历史）+ YAML/JSON（配置）      │
├─────────────────────────────────────────────────────┤
│                 日志与监控层                          │
│  OFLogger / TaskLogger / Diagnostics                │
└─────────────────────────────────────────────────────┘
```

### 各层职责

| 层级 | 职责 | 不做的事 |
|---|---|---|
| UI 展示层 | 渲染界面、接收用户输入、显示数据 | 业务逻辑、OpenFOAM 调用 |
| 应用控制层 | 窗口生命周期、菜单命令路由、全局状态管理 | 具体业务逻辑 |
| 业务逻辑层 | 项目管理、Case 管理、配置生成、结果解析 | UI 渲染、进程启动 |
| OpenFOAM 集成层 | 封装 OpenFOAM 命令、执行、解析输出 | UI 交互 |
| Linux 本地进程调用层 | subprocess 管理、环境变量、进程监控 | 业务语义理解 |
| 文件管理层 | 目录创建/删除、文件读写、路径规范化 | 数据含义理解 |
| 数据存储层 | SQLite CRUD、配置持久化 | 业务规则 |
| 日志与监控层 | 统一日志、任务日志、诊断信息 | 业务逻辑 |

### 层间调用规则

- **上层可调用下层**：UI → 业务逻辑 → OpenFOAM集成 → 进程调用
- **下层不可反向调用上层**：进程调用层通过 Qt 信号回调 UI
- **跨层调用禁止**：业务逻辑层不得直接 import UI 控件

    status TEXT DEFAULT 'idle',
    created_at TEXT NOT NULL,
    FOREIGN KEY (project_id) REFERENCES projects(id)
);

    id TEXT PRIMARY KEY,
    case_id TEXT NOT NULL,
    solver TEXT NOT NULL,
    start_time TEXT NOT NULL,
    end_time TEXT,
    status TEXT,
    log_path TEXT,
    FOREIGN KEY (case_id) REFERENCES cases(id)
);

-- 结果元数据表
CREATE TABLE result_metadata (
    id TEXT PRIMARY KEY,
    case_id TEXT NOT NULL,
    time_label TEXT,
    field_type TEXT,

## I. OpenFOAM 集成设计

### 命令执行原理

```
Python GUI (QThread)
    ↓ Qt 信号
SubprocessRunner.run("bash -lc 'source /opt/openfoam10/etc/bashrc && blockMesh -case /path/to/case'")
    ↓
subprocess.Popen（完全隔离的 bash 环境）
    ↓
OpenFOAM 环境变量正确加载
    ↓
blockMesh / simpleFoam 等命令执行
```

### 关键设计决策

| 问题 | 解决方案 |
|---|---|
| OpenFOAM 环境变量 | 通过 `bash -lc` 执行，让 OpenFOAM bashrc 正确初始化 |
| 命令实时输出 | 通过 `subprocess.PIPE` 捕获 stdout/stderr，通过 Qt 信号实时推送 |
| 多任务隔离 | 每个任务在独立 QThread 中运行，工作目录隔离 |
| 任务中止 | 通过 `process.terminate()` + `process.kill()` 两阶段中止 |
| 命令注入防护 | 所有路径通过 `os.path.normpath()` 规范化，白名单校验路径字符 |

### MVP 阶段 OpenFOAM 支持

| 能力 | MVP 是否支持 |
|---|---|
| blockMesh（结构网格） | ✅ 先做 |
| snappyHexMesh（非结构网格） | ✅ 先做 |
| simpleFoam（稳态不可压缩） | ✅ 先做 |
| pisoFoam（瞬态不可压缩） | ✅ 先做 |
| icoFoam（瞬态层流） | 🔲 后续 |
| reactingFoam（化学反应） | 🔲 后续 |
| paraFoam（结果查看） | ✅ 先做（启动外部 paraFoam） |
| 日志解析（residuals） | ✅ 先做 |
| pyvista 可视化 | ✅ 先做 |

import sys

from ui.main_window import MainWindow

def main():
